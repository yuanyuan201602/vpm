# 四类任务范例

每个范例展示主 Agent 锁定后的产物:**PRD + 验收标准 + 资源/工具清单**。
这就是你要交给制作 Agent 的「交接包」,也是审核 Agent 的依据。

配合 [SPELLS.md](SPELLS.md) 使用。

---

## 范例一 · 写作(一篇公众号文章)

**《PRD》**
- 目标:写一篇讲"AI 帮你做事为什么会跑偏"的科普文章。
- 给谁:非技术的普通读者,刷手机时能读懂。
- 核心价值:读完能记住一个具体方法,而不只是"觉得有道理"。
- 范围:做——1500-2000 字、一个真实例子、一个可操作的建议;不做——技术术语堆砌、参考文献。

**《验收标准》**
1. 开头 100 字内出现一个具体场景,不是抽象道理。
2. 全文出现至少一个"读者今天就能用"的动作。
3. 给一个完全外行读,他能复述出核心观点(不是"写得不错")。
4. 没有"赋能/抓手/闭环"这类空话。

**资源 / 工具**
- 内容资源:你提供 1 个亲身踩坑的例子;主 Agent 可补 1 个公开案例。
- 工具:纯文字 AI 即可。

## 范例二 · 做图(一张产品海报)

**《PRD》**
- 目标:为一款保温杯做一张电商主图。
- 给谁:电商详情页第一屏的潜在买家。
- 核心价值:3 秒内让人看懂"这杯子保温很久"这一个卖点。
- 范围:做——竖版 3:4、突出一个卖点、留出放价格的空白;不做——堆五个卖点、复杂背景。

**《验收标准》**
1. 主卖点文字字号最大,一眼最先看到。
2. 画面有放价格/促销的空白区,没被塞满。
3. 风格干净,不是 AI 那种过度堆砌的廉价感。
4. 杯子本体清晰、不变形。

**资源 / 工具**
- 内容资源:你提供产品照片 + 一句卖点文案。
- 工具:画图工具(如 Midjourney / 即梦 / Nano-Banana 等);文字排版可用图片编辑工具二次处理。

## 范例三 · 视频(一条产品讲解短视频)

**《PRD》**
- 目标:做一条 30-45 秒的保温杯讲解视频。
- 给谁:短视频平台刷到的路人。
- 核心价值:前 3 秒留住人,看完知道这杯子值得买的那一个理由。
- 范围:做——竖屏、有字幕、一个核心卖点;不做——超过 60 秒、复杂转场。

**《验收标准》**
1. 前 3 秒就抛出钩子,不是慢慢介绍。
2. 全程有字幕,关掉声音也能看懂。
3. 结尾有明确的下一步(下单 / 关注)。
4. 整条只讲清一个卖点,不贪多。

**资源 / 工具**
- 内容资源:产品图/片段、卖点文案、背景音乐授权。
- 工具:脚本用文字 AI;配音用 TTS;画面用视频生成或剪辑工具;字幕用剪辑软件。
- (主 Agent 应在清单里把"脚本→配音→画面→字幕"拆成步骤,各配一个工具。)

## 范例四 · 编程(一个小工具)

**《PRD》**
- 目标:做一个把一堆图片批量压缩的小网页工具。
- 给谁:不会用命令行的普通人。
- 核心价值:拖进去图片,点一下,下载压缩好的——全程不用安装任何东西。
- 范围:做——浏览器里直接跑、支持批量、能调压缩质量;不做——账号系统、云端上传。

**《验收标准》**
1. 双击 HTML 文件就能用,无需安装。
2. 一次能处理至少 10 张图。
3. 压缩后能下载,且图片没坏。
4. 不把图片传到任何服务器(本地处理)。

**资源 / 工具**
- 内容资源:几张测试图片(含大文件)用于验收。
- 工具:制作用能写代码的 AI;审核方应**实际打开成品、真的压几张图**来验,而不是只读代码。
