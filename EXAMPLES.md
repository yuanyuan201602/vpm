# 四类任务范例

每个范例展示主 Agent 锁定后的产物:**PRD + 验收标准 + 主动搜集的资源包**。
这就是你要交给制作 Agent 的「交接包」,也是审核 Agent 的依据。

"主动搜集"不是列愿望清单,而是主 Agent 真的去找:**联网搜同类案例、找适合的 Skills/插件/工具**(见 [SPELLS.md](SPELLS.md) 第3步)。下面每个范例的资源块都按"案例参考 + 能力资源 + 内容素材"三件套示范。

配合 [SPELLS.md](SPELLS.md) 使用。

## 范例一 · 写作(一篇公众号文章)

**《PRD》**
- 目标:写一篇讲"AI 帮你做事为什么会跑偏"的科普文章。
- 给谁:非技术的普通读者,刷手机时能读懂。
- 核心价值:读完能记住一个具体方法,而不只是"觉得有道理"。
- 范围:做——1500-2000 字、一个真实例子、一个可操作的建议;不做——技术术语堆砌、参考文献。

**《验收标准》**
1. 开头 100 字内出现一个具体场景,不是抽象道理。
2. 全文出现至少一个"读者今天就能用"的动作。
3. 给一个完全外行读,他能复述出核心观点(不是"写得不错")。
4. 没有"赋能/抓手/闭环"这类空话。

**主 Agent 主动搜集**
- 案例参考:联网搜「AI agent goal drift」「为什么 AI 测试全过产品却错」找 2-3 篇高赞复盘当行文参照;搜「公众号 科普 开头 钩子」拆几个爆款开头结构。
- 能力资源:去 AI 味用 `humanizer-zh` skill;查资料/读链接用联网搜索类工具(如 agent-reach)。
- 内容素材:你提供 1 个亲身踩坑的例子。

## 范例二 · 做图(一张产品海报)

**《PRD》**
- 目标:为一款保温杯做一张电商主图。
- 给谁:电商详情页第一屏的潜在买家。
- 核心价值:3 秒内让人看懂"这杯子保温很久"这一个卖点。
- 范围:做——竖版 3:4、突出一个卖点、留出放价格的空白;不做——堆五个卖点、复杂背景。

**《验收标准》**
1. 主卖点文字字号最大,一眼最先看到。
2. 画面有放价格/促销的空白区,没被塞满。
3. 风格干净,不是 AI 那种过度堆砌的廉价感。
4. 杯子本体清晰、不变形。

**主 Agent 主动搜集**
- 案例参考:联网搜「保温杯 电商 主图 高点击」「product hero shot ecommerce」收集 3 张同类高转化主图,提炼构图/配色规律给制作方。
- 能力资源:出图用 即梦 / Midjourney / Nano-Banana;文字排版二次处理用图片编辑工具(如 Canva / Figma)。
- 内容素材:你提供产品照片 + 一句卖点文案。

## 范例三 · 视频(一条产品讲解短视频)

**《PRD》**
- 目标:做一条 30-45 秒的保温杯讲解视频。
- 给谁:短视频平台刷到的路人。
- 核心价值:前 3 秒留住人,看完知道这杯子值得买的那一个理由。
- 范围:做——竖屏、有字幕、一个核心卖点;不做——超过 60 秒、复杂转场。

**《验收标准》**
1. 前 3 秒就抛出钩子,不是慢慢介绍。
2. 全程有字幕,关掉声音也能看懂。
3. 结尾有明确的下一步(下单 / 关注)。
4. 整条只讲清一个卖点,不贪多。

**主 Agent 主动搜集**
- 案例参考:联网搜「保温杯 短视频 完播率高」「product demo 30s hook」,拆 2-3 条同类爆款的前 3 秒结构给制作方。
- 能力资源:把「脚本→配音→画面→字幕」拆成步骤各配一个工具——脚本用文字 AI、配音用 TTS、画面用视频生成/剪辑、字幕用剪映 / CapCut。
- 内容素材:产品图/片段、卖点文案、背景音乐授权。

## 范例四 · 编程(一个小工具)

**《PRD》**
- 目标:做一个把一堆图片批量压缩的小网页工具。
- 给谁:不会用命令行的普通人。
- 核心价值:拖进去图片,点一下,下载压缩好的——全程不用安装任何东西。
- 范围:做——浏览器里直接跑、支持批量、能调压缩质量;不做——账号系统、云端上传。

**《验收标准》**
1. 双击 HTML 文件就能用,无需安装。
2. 一次能处理至少 10 张图。
3. 压缩后能下载,且图片没坏。
4. 不把图片传到任何服务器(本地处理)。

**主 Agent 主动搜集**
- 案例参考:联网搜「browser image compression library」,找到 browser-image-compression / Compressor.js 这类成熟库,让制作方别从零造轮子;看 1-2 个开源同类工具的界面做参照。
- 能力资源:制作用能写代码的 AI;可装项目自带的 `for-coders/` 套件做确定性闸门。
- 内容素材:几张测试图片(含大文件)用于验收。审核方须**实际打开成品、真的压几张图**,不是只读代码。
