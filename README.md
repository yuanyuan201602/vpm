# 可信化项目管理 · VPM

**一套让 AI 把事做对方向的协作方法——写作、做图、做视频、编程都能用。**

不用懂编程。你需要的只是:三段可以复制粘贴的"咒语",和一个三人 AI 团队的工作方式。

> English background for the framework: [docs/framework-en.md](docs/framework-en.md)

---

## 它要解决的一个问题

AI 最擅长把活干得"**看起来完成了**",也最容易"**做错方向还自信地交给你**"。

一个真实例子(来自一个代码项目,但道理对所有任务都成立):

```
所有技术检查   → 全部通过 ✓
"是不是真做出了要的东西" → 0 分 ✗
```

每一项工程指标都是绿的,产品却完全跑偏。问题不在执行——AI 执行得很好,**它只是在朝错误的目标执行**,而没有人有权叫停。

VPM 就是来堵这个洞:**把"看起来完成"和"真的是我要的"分开,只认后者。**

---

## 核心:两条原则

1. **执行与审核分离。** 干活的 AI,没资格给自己打分。一个 AI 做,**另一个**对照标准独立挑刺。
2. **先把"什么叫做好了"写下来。** 开工前,通过和 AI 讨论,把模糊创意变成一份 PRD(做什么)和一份验收标准(怎么算做好了)。

---

## 工作方式:一个三人 AI 团队

你是**主管**,指挥三个 AI 角色:

| 角色 | 干什么 |
|------|--------|
| **主 Agent**(项目经理) | 陪你讨论 → 写 PRD + 验收标准 → 收集资源和工具 → 分派 → **汇总两边信息做决策** |
| **制作 Agent** | 拿规格和资源,做出成品。不自评 |
| **审核 Agent** | 只拿验收标准 + 成品,逐条挑刺。看不到制作过程 |

关键:**制作和审核从不直接对话**,只通过主 Agent。审核方一旦看到制作方说"我做好了",独立性就被污染了。

---

## 五段流程:从创意到交付

```
① 模糊创意  ─讨论→  ② PRD + 验收标准(你拍板锁定)
                        │
                        ▼
                   ③ 制作 Agent 做
                        │ 成品
                        ▼
                   ④ 审核 Agent 对标准挑刺
                        │ 判定
                        ▼
            主 Agent 汇总 → 通过？
                   ├─ 是 → ⑤ 交付
                   └─ 否 → 退回 ③(带具体问题),或退回②改标准
```

---

## 怎么开始(三步)

1. 打开 [SPELLS.md](SPELLS.md),复制**讨论咒语**,贴给一个 AI,把你的想法说清楚,直到 PRD 和验收标准锁定。
2. 复制**制作咒语** + 锁定的规格,交给制作 AI,拿回成品。
3. **新开一个对话**(或换另一个工具),复制**审核咒语** + 验收标准 + 成品,拿回判定。不达标就带着问题回到第 2 步。

> 单工具也能用:三个角色 = 三个独立对话窗口。唯一不能省的是——**制作和审核必须分开的对话**。

四类任务各有一个填好的范例:[EXAMPLES.md](EXAMPLES.md)(写作 / 做图 / 视频 / 编程)。
